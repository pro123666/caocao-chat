# 操作步骤

## 一、本地开发（自己用）

| 步骤 | 操作 |
|------|------|
| 1 | 进入项目目录：`cd ai-chat-assistant` |
| 2 | 安装依赖（首次）：`npm install` |
| 3 | 在项目根目录创建 `.env`，写入：<br>`VITE_ZHIPU_API_KEY=你的智谱API密钥` |
| 4 | 启动开发服务器：`npm run dev` |
| 5 | 浏览器打开终端里显示的地址（一般是 `http://localhost:5173`）即可使用 |

---

## 二、部署成公开网页（所有人可访问）

### 方式：Vercel

| 步骤 | 操作 |
|------|------|
| 1 | 在 [GitHub](https://github.com) 新建一个仓库（如 `caocao-chat`），不要勾选「Initialize with README」 |
| 2 | 在项目目录打开终端，执行： |
| | `git init` |
| | `git add .` |
| | `git commit -m "曹操智能体"` |
| | `git branch -M main` |
| | `git remote add origin https://github.com/你的用户名/仓库名.git` |
| | `git push -u origin main` |
| 3 | 打开 [vercel.com](https://vercel.com)，用 GitHub 登录 |
| 4 | 点击 **Add New → Project**，选择刚推送的仓库，点击 **Import** |
| 5 | **Framework Preset** 选 **Vite**，**Root Directory** 不动，直接点 **Deploy** |
| 6 | 部署完成后，进入该项目的 **Settings → Environment Variables** |
| 7 | 新增变量：**Name** 填 `ZHIPU_API_KEY`，**Value** 填你的智谱 API Key，点 **Save** |
| 8 | 在 **Deployments** 里对最新一次部署点 **⋯ → Redeploy**，使环境变量生效 |
| 9 | 部署完成后会得到网址，如 `https://xxx.vercel.app`，**把该链接发给别人即可使用**，对方无需配置 Key |

---

## 三、常用命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 本地启动开发服务器 |
| `npm run build` | 打包生产版本（输出在 `dist`） |
| `npm run preview` | 本地预览打包后的效果 |

---

## 四、注意事项

- 本地 `.env` 里的 Key 不要提交到 Git（项目已通过 `.gitignore` 忽略 `.env`）。
- 部署到 Vercel 时，Key 只填在 **Environment Variables**，不要写进代码。
- 部署后若对话报错，先检查 Vercel 里 `ZHIPU_API_KEY` 是否填对，再 **Redeploy** 一次。
