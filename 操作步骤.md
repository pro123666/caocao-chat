# 操作步骤

## 一、本地开发（自己用）

| 步骤 | 操作 |
|------|------|
| 1 | 进入项目目录：`cd ai-chat-assistant` |
| 2 | 安装依赖（首次）：`npm install` |
| 3 | 在项目根目录创建 `.env`，写入：<br>`VITE_ZHIPU_API_KEY=你的智谱API密钥` |
| 4 | 启动开发服务器：`npm run dev` |
| 5 | 浏览器打开终端里显示的地址（一般是 `http://localhost:5173`）即可使用 |

---

## 二、部署成公开网页（所有人可访问）

### 方式：Vercel

| 步骤 | 操作 |
|------|------|
| 1 | 在 [GitHub](https://github.com) 新建一个仓库（如 `caocao-chat`），不要勾选「Initialize with README」 |
| 2 | 在项目目录打开终端，执行： |
| | `git init` |
| | `git add .` |
| | `git commit -m "曹操智能体"` |
| | `git branch -M main` |
| | `git remote add origin https://github.com/你的用户名/仓库名.git` |
| | `git push -u origin main` |
| 3 | 打开 [vercel.com](https://vercel.com)，用 GitHub 登录 |
| 4 | 点击 **Add New → Project**，选择刚推送的仓库，点击 **Import** |
| 5 | **Framework Preset** 选 **Vite**，**Root Directory** 不动，直接点 **Deploy** |
| 6 | 部署完成后，进入该项目的 **Settings → Environment Variables** |
| 7 | 新增变量：**Name** 填 `ZHIPU_API_KEY`，**Value** 填你的智谱 API Key，点 **Save** |
| 8 | 在 **Deployments** 里对最新一次部署点 **⋯ → Redeploy**，使环境变量生效 |
| 9 | 部署完成后会得到网址，如 `https://xxx.vercel.app`，**把该链接发给别人即可使用**，对方无需配置 Key |

## 三、国内 CDN 加速（页面走国内，接口直连 Vercel）

**效果**：国内用户访问国内域名，页面从 CDN 加载，对话接口直连 Vercel，无需 CDN 支持 POST/流式。

### 3.1 在 Vercel 已部署好的前提下

| 步骤 | 操作 |
|------|------|
| 1 | 记下你的 Vercel 地址，例如：`https://caocao-chat-xxx.vercel.app` |
| 2 | 在项目目录执行（**把下面地址换成你的 Vercel 地址**）：<br>Windows PowerShell：<br>`$env:VITE_VERCEL_ORIGIN="https://caocao-chat-xxx.vercel.app"; npm run build`<br>或 CMD：<br>`set VITE_VERCEL_ORIGIN=https://caocao-chat-xxx.vercel.app && npm run build` |
| 3 | 得到 `dist` 目录后，把 **整个 dist 里的内容** 上传到又拍云/七牛等「静态托管」或「对象存储 + CDN」，并绑定你的**国内域名**（需备案） |
| 4 | 国内用户访问 `https://你的国内域名` 即可：页面从国内 CDN 取，发消息直连 Vercel |

说明：项目已支持跨域，Vercel 上的 `/api/chat` 已允许被任意域名调用。

### 3.2 又拍云快速操作（静态托管）

| 步骤 | 操作 |
|------|------|
| 1 | 登录 [又拍云控制台](https://console.upyun.com) |
| 2 | 进入 **存储** → **云存储**，创建空间（如 `caocao-chat`），选「静态网站」 |
| 3 | 在空间里**上传**本机 `dist` 目录下的全部文件（`index.html` 在根目录，`assets` 等保持原样） |
| 4 | 在空间设置里绑定**加速域名**（你的国内域名），并到域名服务商处把该域名 CNAME 到又拍给的地址 |
| 5 | 访问 `https://你的国内域名` 测试：能打开页面且能发消息即成功 |

## 四、常用命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 本地启动开发服务器 |
| `npm run build` | 打包生产版本（输出在 `dist`） |
| **国内 CDN 用** | `$env:VITE_VERCEL_ORIGIN="https://你的项目.vercel.app"; npm run build`（PowerShell） |
| `npm run preview` | 本地预览打包后的效果 |

---

## 五、注意事项

- 本地 `.env` 里的 Key 不要提交到 Git（项目已通过 `.gitignore` 忽略 `.env`）。
- 部署到 Vercel 时，Key 只填在 **Environment Variables**，不要写进代码。
- 部署后若对话报错，先检查 Vercel 里 `ZHIPU_API_KEY` 是否填对，再 **Redeploy** 一次。
